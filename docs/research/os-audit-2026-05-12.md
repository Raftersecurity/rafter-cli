## 1. Consumer / UX

*First-time user experience: onboarding, error messages, help text, discoverability.*

### HIGH

**UX-1 — Misleading `rafter agent init --all` in quickstart**  
README.md:50-54. Quickstart says the command "Installs all detected integrations and Downloads Betterleaks" but on a machine with no agent tools installed, nothing happens except config init — with a message "No agent environments detected." New users follow the quickstart and see no result.  
*Fix:* Add a prerequisite note: "Requires at least one agent tool (Claude Code, Cursor, etc.) to be installed first."

**UX-2 — Confusing fallback message when no agents detected**  
`node/src/commands/agent/init.ts:1443-1456`. The output distinguishes "no integrations installed" vs "no agent environments detected" — two messages new users can't distinguish. Contradicts the earlier "Agent security initialized!" success message.  
*Fix:* Consolidate into one clear message with a call-to-action: "Install Claude Code, Cursor, or another agent, then re-run `rafter agent init --all`."

**UX-3 — `rafter agent init` without flags has no visible output**  
First-time users running bare `rafter agent init` see no feedback on what happened. The two-step flow (init → init --all) is undocumented.  
*Fix:* Print a next-steps prompt or add an interactive mode that runs by default.

### MEDIUM

**UX-4 — Update notice shows npm command first even for pip users**  
`node/src/utils/update-checker.ts:103`. Shows npm command before pip regardless of install method.  
*Fix:* Detect install method and surface the matching command first.

**UX-5 — Missing alternative install methods in README**  
README.md:93-109. No mention of `npx`, Homebrew, Docker, or direct binary download.  
*Fix:* Add "Alternative Installation Methods" section.

**UX-6 — Engine flag error message lacks context**  
`node/src/commands/agent/init.ts:1071-1078`. "Invalid engine: X. Valid values: auto, betterleaks, patterns" — doesn't explain when to choose each.  
*Fix:* Expand: "betterleaks = enhanced patterns (21+ rules), patterns = built-in regex, auto = auto-select."

**UX-7 — Pre-commit hook block message truncates commands**  
`node/src/commands/hook/pretool.ts:32`. Truncates at 60 chars. User can't tell which arg triggered the block.  
*Fix:* Print full command (let terminal wrap) or reference `rafter agent audit --last 1`.

**UX-8 — `--local` flag undocumented use-case**  
README.md:184. Mentions `--local` without explaining when to use it (CI, Docker, ephemeral environments).  
*Fix:* Add sidebar: "Use --local for ephemeral environments — keeps config in ./.rafter/ instead of ~/.rafter/."

**UX-9 — `--dry-run` help text has internal reference tag**  
`node/src/commands/agent/init.ts:1051`. Help text contains `(rf-hrtd)` — an internal tag exposed to users.  
*Fix:* Remove the tag; rewrite the description to describe exit codes.

### LOW

**UX-10 — Warning emoji may not render in all terminals**  
`node/src/utils/formatter.ts:26`. `⚠️` (emoji variant) fails in some CI consoles and older Windows terminals.  
*Fix:* Use `⚠` (U+26A0, text variant) or `[WARN]`.

**UX-11 — `--quiet` interaction with `--json` is ambiguous**  
README.md:202. Doesn't clarify that `--quiet` only suppresses stderr, not JSON stdout.  
*Fix:* Document: "--quiet suppresses status/progress output to stderr; JSON output is unaffected."

**UX-12 — "Agent security initialized!" misleads when nothing was installed**  
`node/src/commands/agent/init.ts:1422`. Success message followed by "No integrations were installed" is contradictory.  
*Fix:* Change to conditional: "Config created. No integrations installed yet — run with --all."

**UX-13 — Python README shows deprecated command names**  
`python/README.md:43`. Shows `rafter agent scan .` without deprecation notice. Main README uses `rafter secrets`.  
*Fix:* Add deprecation note; update examples to `rafter secrets`.

**UX-14 — API key troubleshooting missing from README**  
README.md:150-155. No guidance on testing key validity or locating the key in the dashboard.  
*Fix:* Add: "Test with `rafter usage` (exit 0 = valid). Invalid keys show: 'Error: Unauthorized'."

**UX-15 — No PATH warning after install if `rafter` not found**  
`node/src/commands/agent/init.ts:1063-1068`. No check whether `rafter` is on PATH after global install.  
*Fix:* After init, run `which rafter`; if missing, warn with reinstall instructions.

## 2. AI-Generated Code Quality

*Signs of insufficiently reviewed AI-generated code: naming inconsistency, dead code, over-engineering.*

### HIGH

**AIQ-1 — 78 instances of `: any` defeating TypeScript safety**  
`node/src/utils/skill-manager.ts:143`, `node/src/commands/agent/components.ts:74,80`, and ~75 more. Critical paths like JSON serialization and config merging use `any` instead of `Record<string, unknown>` or typed interfaces.  
*Fix:* Replace with specific types; use `unknown` + type guards at runtime boundaries.

**AIQ-2 — Inconsistent error handling in Python: silent swallow vs warn vs raise**  
`python/rafter_cli/commands/agent.py:400-405,435,515`. Lines 404/486 catch `Exception` silently; lines 435/449 warn and continue; lines 515/517 re-raise. No consistent policy.  
*Fix:* Establish a policy: always catch specific exceptions, always log what happened, decide per-function whether to warn+continue or raise.

**AIQ-3 — Catch-all `except Exception:` with misleading message**  
`python/rafter_cli/commands/agent.py:400-405`. Catches all exceptions (permission denied, OOM, KeyboardInterrupt) with a message that only applies to file-not-found.  
*Fix:* `except (FileNotFoundError, KeyError):` with targeted messages per exception type.

### MEDIUM

**AIQ-4 — Inconsistent naming: `Finding` vs `Result` vs `Match`**  
`node/src/scanners/betterleaks.ts:17` (`BetterleaksResult`), `node/src/scanners/regex-scanner.ts:7` (`ScanResult`), `node/src/core/custom-patterns.ts:118` (`SuppressedFinding`). Same concept named three ways.  
*Fix:* Standardize to `Finding` as the unit and `ScanResult` as the envelope.

**AIQ-5 — Over-engineered formatter: 6 near-identical wrapper functions**  
`node/src/utils/formatter.ts:13-55`. Six functions each containing one if-statement on `agentMode`. Could be a single `format(type, text)` function.  
*Fix:* Consolidate to a switch/map on type.

**AIQ-6 — Duplicated engine-selection logic across scan functions**  
`python/rafter_cli/commands/agent.py:1411-1444`. `_scan_file` and `_scan_directory` both contain the same try-betterleaks/fall-back-to-patterns logic.  
*Fix:* Extract to `_get_scanner(engine: str)` helper.

**AIQ-7 — Repetitive if-block (8 identical lines) for platform hints**  
`node/src/commands/agent/init.ts:1446-1453`. Eight `if (hasPlatform) console.log(...)` lines that should be a loop over a `{has, name, flag}` array.  
*Fix:* Replace with `platforms.forEach(p => { if (p.has) console.log(...) })`.

**AIQ-8 — Incomplete/broken UI string**  
`node/src/commands/agent/init.ts:1436`. Message ends with "from .aider.conf.yml read:" — sentence fragment.  
*Fix:* `"Restart Aider to load RAFTER.md from ~/.aider.conf.yml"`.

**AIQ-9 — Tests check existence rather than behavior**  
`node/tests/error-handling-gauntlet.test.ts:582-588`. Uses `toBeDefined()` and `toContain(severity)` instead of verifying the actual match content is correct.  
*Fix:* Assert `expect(m.match).toMatch(m.pattern.regex)` and `expect(m.line).toBeGreaterThan(0)`.

### LOW

**AIQ-10 — Dead parameter: `cursor` unused but kept "for call-site compatibility"**  
`python/rafter_cli/commands/agent.py:465`. `_ = cursor  # parameter kept for call-site compatibility` — a forgotten refactoring artifact.  
*Fix:* Remove the parameter or document the actual reason it's kept.

**AIQ-11 — Awkward function signature mixing positional + default params**  
`python/rafter_cli/commands/agent.py:408-415`. `_install_global_instructions` adds `windsurf=False` at the end of a mostly positional signature.  
*Fix:* Either go fully keyword-only or accept a config dict.

**AIQ-12 — Obvious comments that restate code**  
`node/src/commands/agent/config.ts:9,19,40,49`. Comments like `// Show all config`, `// Get specific value`, `// Try to parse value as JSON, otherwise use as string`.  
*Fix:* Remove these; keep only "why" comments.

**AIQ-13 — Dead code: `unused` parameter comment**  
Several places where unused parameters are commented with notes that should have been refactored out.  
*Fix:* Remove or document properly.

## 3. Security of the Security Tool

*Security vulnerabilities in a security tool are a credibility issue. Ordered by severity.*

### HIGH

**SEC-1 — Command injection via string interpolation in `git.ts`**  
`node/src/utils/git.ts:3-4`. `execSync(\`git ${cmd}\`)` concatenates unvalidated input into a shell string. The function is exported and public.  
*CWE-94. Fix:* Use array-form `execSync` or parse arguments explicitly.

### MEDIUM

**SEC-2 — Path traversal via config-supplied log path**  
`node/src/core/audit-logger.ts:244-252` + `node/src/core/config-manager.ts:306-307`. `.rafter.yml` can set `audit.logPath` to `../../../etc/cron.d/backdoor` — loaded without restriction.  
*CWE-22. Fix:* Validate resolved path is under `~/.rafter/`; reject traversals.

**SEC-3 — Shell injection in git global hook installation**  
`node/src/commands/agent/install-hook.ts:118`. `execSync(\`git config --global core.hooksPath "${globalHooksDir}"\`)` — home directories with special characters could inject.  
*CWE-94. Fix:* Use array-form `execSync`.

**SEC-4 — Shell injection in Gemini skills registration**  
`node/src/commands/agent/init.ts:1001`. `execSync(\`gemini skills link ${JSON.stringify(absPath)}\`)` — still uses string interpolation.  
*CWE-94. Fix:* `execSync('gemini', ['skills', 'link', absPath])`.

**SEC-5 — Python tarball extraction may not reject symlinks**  
`python/rafter_cli/utils/binary_manager.py`. Node's `binary-manager.ts:566-592` has explicit symlink rejection; Python needs verification.  
*CWE-59. Fix:* Audit Python extraction path; add explicit symlink check.

**SEC-6 — MCP `get_doc` path traversal via `id_or_tag`**  
`node/src/commands/mcp/server.ts:218-240`. MCP client controls `id_or_tag` which is passed to `resolveDocSelector()`. If that resolves file paths without validation, path traversal is possible.  
*CWE-22. Fix:* Validate selector input before file-system resolution.

### LOW

**SEC-7 — Audit log doesn't redact secrets that don't match any pattern**  
`node/src/core/audit-logger.ts:390-412`. Commands are redacted via regex, but unmatched secrets log unredacted.  
*CWE-532. Fix:* Document clearly; encourage strict log file permissions (already 0o600).

**SEC-8 — Betterleaks temp file uses predictable naming**  
`node/src/scanners/betterleaks.ts:90`. `Date.now()` has limited entropy; a race could pre-create a symlink.  
*CWE-367. Fix:* Use `fs.mkdtempSync()`.

**SEC-9 — Version parameter validated in Node but needs verification in Python**  
`node/src/utils/binary-manager.ts:202-204` validates with `VERSION_PATTERN`. Python equivalent should be checked.  
*Fix:* Confirm `python/rafter_cli/utils/binary_manager.py` has equivalent validation.

**SEC-10 — SSRF validation exists but Python flow needs end-to-end verification**  
`python/rafter_cli/core/audit_logger.py:38-78`. Node side looks correct. Python audit logger notification flow should be traced to confirm validation is always called.

## 4. Performance & Efficiency

*CLI tools run on every commit. Startup time and scan speed matter.*

### HIGH

**PERF-1 — Regex patterns recompiled per file (critical hot path)**  
`node/src/core/pattern-engine.ts:128-146`. `new RegExp()` called for every pattern × every file scanned. 21 patterns × 100 files = 2,100 recompilations per `rafter secrets .`.  
*Estimated impact: 100–500 ms per scan. Fix:* Pre-compile in constructor; use cached instances.

**PERF-2 — Same issue in Python**  
`python/rafter_cli/core/pattern_engine.py:108-117`. `re.compile()` called in every `scan()` iteration.  
*Fix:* Compile once in `__init__`, store in instance.

**PERF-3 — MCP server reinitializes scanners/interceptors on every tool call**  
`node/src/commands/mcp/server.ts:141-200`. `BetterleaksScanner()`, `RegexScanner()`, `CommandInterceptor()`, and `AuditLogger()` created fresh for each tool invocation. Pattern loading + file I/O on every call.  
*Fix:* Create instances once in `createServer()`; reuse across calls.

**PERF-4 — Python GIL blocks parallel file scanning**  
`python/rafter_cli/scanners/regex_scanner.py:57-63`. Serial file iteration with no asyncio or multiprocessing. On a 4-core machine, this is 3–4× slower than parallel.  
*Fix:* Use `concurrent.futures.ThreadPoolExecutor` for I/O-bound file reading.

### MEDIUM

**PERF-5 — No file size limit; large binary files stall scan**  
`node/src/scanners/regex-scanner.ts:33-40`. `fs.readFileSync(filePath, "utf-8")` reads entire file regardless of size. Extension-based binary filter doesn't catch `.log` files.  
*Fix:* Add configurable size limit (default 10 MB); skip oversized files with a warning.

**PERF-6 — Synchronous, single-threaded directory walking**  
`node/src/scanners/regex-scanner.ts:128-165`. Fully synchronous recursive walk. On 100,000-file repos, takes 1–5 seconds.  
*Fix:* Use async `fs.promises.readdir()` with concurrency control, or `fast-glob`.

**PERF-7 — `redactText()` recompiles regexes on every call**  
`node/src/core/pattern-engine.ts:84-95`. Same compile-per-call problem in the redaction path.  
*Fix:* Covered by PERF-1 fix (use pre-compiled instances).

**PERF-8 — Custom patterns reloaded from disk on every scanner instantiation**  
`node/src/core/custom-patterns.ts:25-51`. Reads `.rafter/patterns/` directory every time `RegexScanner` is constructed.  
*Fix:* Module-level cache with TTL or `fs.watch()` invalidation.

### LOW

**PERF-9 — `dotenv.config()` at top-level startup**  
`node/src/index.ts:25`. Reads `.env` on every invocation, even `rafter --help` or `rafter version`.  
*Fix:* Lazy-load only in commands that need it.

**PERF-10 — Update checker blocks process exit on slow networks**  
`node/src/index.ts:100-102`. Non-blocking `.then()` but keeps event loop open. Users see 2–5 second hang on slow networks.  
*Fix:* Call `.unref()` on the underlying Promise/timer so it doesn't prevent exit.

**PERF-11 — Betterleaks PATH check runs on every scan if managed binary absent**  
`node/src/scanners/betterleaks.ts:56-75`. `execSync("which betterleaks")` called per scan when managed binary not found.  
*Fix:* Cache PATH lookup result with TTL.

**PERF-12 — Glob matching for suppressions not cached**  
`node/src/core/custom-patterns.ts:269-277`. `minimatch()` called per finding × per suppression rule. 100 suppressions × 50 findings = 5,000 calls.  
*Fix:* Pre-compile `minimatch.Minimatch` objects; cache by rule.

## 6. Dual-Implementation Sync Debt

*Where Node.js and Python have diverged from feature parity.*

**Overall parity is strong**: commands, secret patterns (23), config schema, risk rules, MCP tools (6), exit codes, and platform support (8) are all identical.

### HIGH

**SYNC-1 — `-V` short flag: advertised in Node completion but doesn't work**  
`node/src/commands/completion.ts:283` advertises `-V` for version; `node/src/index.ts:39` only registers `--version`. Python correctly supports both.  
*Fix:* Add `.option('-V')` to Commander registration, or remove `-V` from completion script.

**SYNC-2 — `--interactive` / `-i` flag missing from Python**  
`node/src/commands/agent/init.ts:1042` supports `-i, --interactive`. Python has no equivalent. Documented as "Node only" in CLI_SPEC.md but not prominently.  
*Fix:* Implement in Python with `typer.prompt()`, or formally document as Node-only in spec and README.

### MEDIUM

**SYNC-3 — Node completion scripts missing several flags for agent scan/init**  
`node/src/commands/completion.ts:54,70`. Bash completion for `agent scan` omits `--format`, `--engine`, `--baseline`, `--watch`, `--history`. `agent init` omits `--local`, `--dry-run`, `--interactive`, `--update`.  
*Fix:* Regenerate or manually update completion.ts to include all current flags.

### LOW

**SYNC-4 — Python has 47 test files vs Node's 64**  
Node has 17 more test files including: `agent-compatibility.test.ts`, `cross-runtime-parity.test.ts`, `platform-integration.test.ts`, `skill-review.test.ts`.  
*Fix:* Port coverage gaps to Python (see Test Coverage angle for specifics).

**SYNC-5 — Python risk rules differ slightly in chain-operator handling**  
`python/rafter_cli/core/risk_rules.py`. Node's `assess_command_risk()` checks `CHAIN_OPERATORS` for `SAFE_PREFIX` exemption; Python doesn't apply the same check.  
*Fix:* Verify and align chain-operator logic between implementations.

**SYNC-6 — Hook completion only advertises `pretool`; `posttool` missing**  
`node/src/commands/completion.ts:37`. Bash completion for `hook` subcommand only shows `pretool --help`; `posttool` is missing.  
*Fix:* Add `posttool` to completion list.

**SYNC-7 — Test file naming conventions differ (expected but noted)**  
Node uses camelCase `.test.ts`; Python uses `test_snake_case.py`. Not a parity issue but noteworthy for contributors.

## 7. Test Coverage Quality

*Coverage gaps, quality issues, flakiness.*

### HIGH

**TEST-1 — 10 agent subcommands lack dedicated integration tests**  
Missing tests for: `agent install-hook`, `agent instruction-block`, `agent update-betterleaks`, `agent baseline`, `agent components`, `agent init-project`, `agent audit`, `agent audit-skill`, `agent verify`, `agent scan` (deprecation path).  
*Fix:* Add dedicated test files for each; cover success path, error cases, and idempotency.

**TEST-2 — Docs commands have no CLI-level tests**  
`node/src/commands/docs/` (list.ts, show.ts) are tested only at the loader level, not as CLI commands.  
*Fix:* Create `node/tests/docs-commands.test.ts` covering `docs list` and `docs show` with/without cache.

### MEDIUM

**TEST-3 — Timing-dependent tests create flakiness**  
`node/tests/notifications.test.ts:88-89,121,141` and `audit-logger-lifecycle.test.ts` use arbitrary 50–100 ms delays.  
*Fix:* Use `vi.waitFor()` or event-based waiting; add meaningful timeout messages.

**TEST-4 — Insufficient false-positive tests for secret patterns**  
`node/tests/secret-patterns.test.ts`. Only 7 negative cases across 20+ pattern types. No off-by-one, placeholder value, or log-excerpt tests.  
*Fix:* Expand with placeholder values (`changeme`, `password123`), log excerpts that match prefixes, and case variations.

**TEST-5 — Weak assertions across test suite (256 instances)**  
`.toBeGreaterThan(0)` instead of exact counts, `.toBeDefined()` instead of value checks, `.toBeTruthy()` instead of `.toBe(true)`.  
*Fix:* Add ESLint rule banning loose matchers; fix critical instances like `expect(critical.length).toBeGreaterThanOrEqual(21)`.

**TEST-6 — Platform-specific coverage only tests current platform**  
`node/tests/platform-integration.test.ts`. Doesn't mock `process.platform`/`process.arch` for all 8 supported combinations.  
*Fix:* Parametrize binary-manager tests for darwin-x64, darwin-arm64, linux-x64, linux-arm64, windows-x64; verify correct download URL and error messages per platform.

**TEST-7 — No network isolation for webhook URL tests**  
`node/tests/notifications.test.ts:52`. `validateWebhookUrl("https://hooks.slack.com/...")` performs actual DNS resolution. Fails offline.  
*Fix:* Mock `dns.resolve` in all webhook URL tests.

**TEST-8 — Hook installation not tested end-to-end**  
`node/tests/hook-formats.test.ts` verifies JSON/YAML output but not actual IDE settings modification, idempotency, or hook execution.  
*Fix:* Create `node/tests/hook-installation-e2e.test.ts` with fake `.git/hooks` dir; verify hook creation, permissions, and idempotency.

**TEST-9 — Command interceptor policy combinations not tested**  
`node/tests/command-interceptor-policy.test.ts`. No tests for: multiple-rule conflicts (which risk level wins?), empty policy, contradictory rules.  
*Fix:* Add `describe("Policy conflict resolution")` block covering multi-rule scenarios.

### LOW

**TEST-10 — Concurrency safety test doesn't verify under true race conditions**  
`node/tests/audit-logger-lifecycle.test.ts`. Concurrent write test simulates but doesn't fire truly concurrent writes.  
*Fix:* `Promise.all(Array(50).fill().map(() => logger.log(...)))` then verify 50 valid JSON lines.

**TEST-11 — Symlink edge cases too limited**  
`node/tests/regex-scanner.test.ts`. Only 2 symlink tests; missing symlink chains, broken symlinks, relative vs absolute.  
*Fix:* Expand with chain symlinks (a→b→c), broken symlinks, intra-directory symlinks.

**TEST-12 — No edge-case tests for large/boundary inputs**  
No tests for files >100 MB, 10,000+ matches in one file, or directory depth >20.  
*Fix:* Add to `secret-scanning-e2e.test.ts`; generate large fixtures; verify memory limits.

**TEST-13 — Betterleaks severity tests too thin**  
`node/tests/betterleaks-severity.test.ts`. Only 3 cases; no unknown severity codes, version-specific changes, or multi-severity files.  
*Fix:* Parametrize with all known betterleaks rule codes.

**TEST-14 — Custom pattern validation doesn't catch catastrophic regex**  
`node/tests/custom-patterns.test.ts`. No test for patterns like `(a+)+b` that cause exponential backtracking.  
*Fix:* Add detection + rejection for known-bad backtracking patterns.

**TEST-15 — Docs cache TTL edge cases untested**  
`node/tests/docs-loader.test.ts`. No test for TTL=0, negative TTL, or timezone-aware expiry.  
*Fix:* Add test for cache invalidation at exact TTL boundary.
